# 주문하기 서비스 로직 (장바구니 기반)

이번 학습은, 그냥 간단하게 장바구니 기반 서비스를 분석한다.

1. 장바구니 정보 조회
2. 장바구니에 담긴 상품 정보 조회(재고량, 가격, 판매 유무 등)
   - 장바구니에 흰티, 청바지가 9월 1일까지 존재했는데 주문을 한 9월 3일날 진행 함.
     이 사이에 재고가 없을 수 있고, 가격이 달라질 수 있음. 그리고 판매를 이때까지 9월 2일까지 마무리하고, 9월 3일까지 안한데(회사 정책상)
     이럴 수 있음. 이럴 경우가 생길 수 있기 때문에 주문하기 할 때 장바구니에 담긴 상품이 실제 주문이 가능한 상품인가를 한 번더 validation을 해줘야 함.
3. 주문상세 정보 생성
   - 장바구니에 담긴, 흰티와 청바지는 각각의 주문 상세로 들어감.
   - 주문 상세 데이터가 2개가 될 꺼임. (1, 2)
4. 주문 정보 생성
   - 주문 상세 정보를 바탕으로 주문 정보 Header의 역할임.
   - 이미 주문한 것에 대한 장바구니 내역은, 또 그다음의 주문을 할 수 있겟지만, 당장 주문을 하지 않음.
   - 나중에 주문하더라도 주문 내역 기반으로, 재 주문하기라는 기능을 넣으면 됨.
5. 장바구니 정보 삭제
   - 우선은 카드에 담긴 정보를 내가 계속해서 주문 할 일은 없기 떄문에 이 장바구니 정보를 삭제
   - 외적으로 추가적인 Validation이 있으면 걷어주면 됨 .

기타) 적절한 곳에서 validation 필요.

테스트 코드는 Given When Then 패턴으로 작성한다.
